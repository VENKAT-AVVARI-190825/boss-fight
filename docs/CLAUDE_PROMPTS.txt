  with fields: countryName, currencyCode, currencyName."

PROMPT 4 (Exchange Rate API):
  "Create GET /exchange-rate/{fromCur}/{toCur} that:
  1. Returns direct rate if stored
  2. Computes inverse if only reverse direction stored
  3. Bridges via INR for pairs not directly connected
  4. Validates currencies — only INR, USD, CAD, EUR, AUD, AED allowed"

COMMANDS USED:
  - Implemented route with three-tier lookup: direct → inverse → INR bridge
  - Pydantic v2 ExchangeRateRequest schema validates and normalises codes
  - Returns 400 for invalid/unsupported currencies, 404 for unavailable pairs
